## 1. Why this plan exists

T7.1 shipped section-aware slicing and spec-aligned naming. However, creating a
specific pack type (drum hits, loops, acapella, phrases) still required manually
setting `--mode sample`, `--segment-beats`, `--stem`, `--format`, and `--fx` for
each use case. This plan adds a preset registry so users can do:

    toolshop remix song.wav --preset drum-kit --stems-dir ./stems/...

and get a labeled pack with a `PACK_README.md` and a `preset` field in the manifest.

## 3. Design

### 3.1 Preset Registry (`remix_adapter.py`)

```python
PACK_PRESETS = {
    "drum-kit": {"mode": "sample", "stem_name": "drums", "segment_beats": 1, "default_label": "drum", "output_format": "wav"},
    "loop-kit": {"mode": "sample", "segment_beats": 4, "default_label": "loop", "output_format": "flac"},
    "acapella-kit": {"mode": "sample", "stem_name": "vocals", "segment_beats": 2, "default_label": "vocal", "output_format": "wav"},
    "remix-kit": {"mode": "sample", "segment_beats": 8, "default_label": "phrase", "output_format": "flac", "fx_chain": ["compressor"]},
}
```

`get_preset(name)` returns a copy; raises `ValueError` for unknown names.

### 3.2 `--preset` CLI Flag

- Added to remix subparser with `choices=list(PACK_PRESETS.keys())`.
- `_apply_preset(args)` in `remix_cli.py` fills in defaults for `mode`, `segment_beats`, `format`, `stem`, `fx` when user didn't explicitly set them.
- `--preset` forces `--mode sample`; combining with `--mode remix` raises `ValueError`.
- `preset_name` passed to `create_remix`.

### 3.3 PACK_README Generator

`_write_pack_readme(output_dir, preset_name, src_bpm, src_key, target_bpm, target_key, fx_chain, samples)`
writes `PACK_README.md` with:
- Pack name (directory name)
- Preset, generated timestamp, source BPM/key, target BPM/key, FX
- Sample count
- Markdown table of samples (filename, section, start–end, BPM)
- Footer: `Generated by toolshop remix --preset <name>`

### 3.4 Manifest + RemixResult

- `_sample_manifest` gets `preset_name` param; adds `"preset"` key when not None.
- `RemixResult` gets `preset: Optional[str] = None` field.

## 6. Risks / notes
- Preset + explicit flags: explicit flag always wins (tested).
- `--preset` + `--mode remix` → `ValueError` (tested).
- `PACK_README.md` is additive — no existing file conflict.
- Argparse defaults changed to `None` for `--mode`/`--fx`/`--segment-beats` so preset values can fill in. No user-visible change for existing commands (`_apply_preset` fills in the old defaults when no preset is used).
